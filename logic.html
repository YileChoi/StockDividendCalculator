Calculation Logic

This page defines the exact accounting mechanics used by the app so all family members can verify fairness and audit outcomes.

Back To Calculator

1) Economic Method

The app uses a unitized fund model (the same method commonly used in pooled investment accounting):

  • Deposits buy units at the current NAV.
  • Withdrawals redeem units at the current NAV.
  • Profit/loss changes NAV for everyone proportionally.
  • People who joined later do not receive profit generated before they entered.

2) Core Formulas

NAV = Portfolio Value / Total Units

Units Bought = Deposit Amount / NAV

Units Redeemed = Withdrawal Amount / NAV

Member Equity = Member Units x NAV

Member Net P/L = Total Withdrawn + Current Equity - Total Contributed

3) Precision And Anti-Error Controls

  • Cash is stored as integer cents.
  • Units are stored with 1e-8 unit precision.
  • Deposits use conservative unit rounding to avoid accidental over-allocation.
  • Withdrawals use conservative rounding so redeemed units always cover requested cash.
  • Invariants are checked after each transaction:
  • Sum(member units) must equal total units.
  • No negative balances are allowed.
  • No portfolio value can exist without owners.
  • Every event is recorded in transaction history for audit.

4) Worked Example

Person A deposits $10,000 at NAV $1.0000, so A receives 10,000 units.

Portfolio gains $2,000. NAV becomes $1.2000. A still has 10,000 units and now holds $12,000 equity.

Person B later deposits $6,000 at NAV $1.2000, so B receives 5,000 units.

Total units are now 15,000. Ownership is A: 66.67%, B: 33.33%.

This is fair because B does not claim A's earlier $2,000 gain.
